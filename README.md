# Portfolio Setup Task
 
This repository is part of the setup task for the AI Operations role. Below is what I installed, the steps I completed, and notes on how it went.
 
## Tools I Installed
 
- Cursor IDE (Windows)
- Claude Code extension (by Anthropic)
- Codex extension (by OpenAI)
- Git + a GitHub account (this repository)
## Steps I Completed
 
1. Downloaded Cursor from cursor.com, ran the installer on Windows, and signed in with my email.
2. Opened the Extensions panel in Cursor (Ctrl+Shift+X), searched for "Claude Code", installed the official Anthropic extension, and signed in with my Claude account through the browser prompt.
3. Installed the official OpenAI "Codex" extension the same way and signed in with my OpenAI account.
4. Created a GitHub account, then created this public repository with a README file.
5. Cloned the repository into Cursor (Git: Clone with the HTTPS link), edited this README, saved it, then committed and pushed it back to GitHub from the Source Control panel.
## Issues I Ran Into and How I Solved Them
 
I didn't hit any blocking errors, but a few things were new to me and worth noting:
 
- **Git terms were new.** I wasn't sure about the difference between "commit" and "push" at first. I learned that commit saves a snapshot locally and push uploads it to GitHub, and that I need both for changes to show up online.
- **Extension sign-in flow.** Signing in to the Claude Code and Codex extensions opens a browser window and then returns to Cursor. The first time, I wasn't sure it had worked, so I confirmed by checking that the extension showed me as logged in before moving on.
- **When unsure, I used AI to guide me.** Instead of guessing, I used Claude to explain each step before I did it myself. That's the same workflow this role describes — using AI tools and docs to get unstuck — so I applied it here.
## What I Learned
 
- How to set up a full AI-assisted coding environment (Cursor + Claude Code + Codex) from scratch.
- The basic GitHub workflow: create a repo, clone it, edit, commit, push.
- That most setup problems are solved fastest by reading the prompt on screen carefully and asking AI or the docs before clicking randomly.
