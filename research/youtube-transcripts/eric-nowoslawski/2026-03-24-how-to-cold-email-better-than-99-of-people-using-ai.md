## Key takeaways
- Stop relying on structured data (e.g., LinkedIn "new in role" dates) because it's the same for everyone — use Claygent to pull unstructured, company-specific research instead.
- Use a one-two punch: Claygent does only the research, then a separate AI integration writes the message — never ask one agent to do both.
- Example workflow: prompt Claygent to check if a prospect's website publicly lists pricing, summarize it, then have a downstream prompt convert that into a line like "I saw on your pricing page your professional plan is $175/month… I'd imagine your LTV is $2,000-$5,000."
- Let Clay auto-generate the Claygent prompt from a short instruction, then review it before accepting; Eric uses GPT-4o Mini as his agency's daily-driver model (not Clay's Neon default).
- Give the message-writing prompt heavy context and hand-written examples — dictate context with Whisperflow (speech-to-text) but manually type the examples.
- Spot-check Claygent outputs against a few live websites and, when it misses, teach it where you had to look manually so it improves.
