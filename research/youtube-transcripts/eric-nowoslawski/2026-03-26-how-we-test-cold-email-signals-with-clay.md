## Key takeaways
- Run three parallel campaign structures to isolate signal value: one broad TAM campaign, one segmented (vertical + headcount + department), and one built around high-confidence signals like "new in role" or "just announced new product."
- Instead of paying to enrich every prospect with expensive signals up front, route every positive reply into a Clay table and run the full signal checklist (Claygent for news, BuiltWith for tech installs, AE-to-SDR ratio, etc.) only on responders.
- Also enrich negative responses with the same checklist so you can compare what's true about companies that say yes vs. no in the moment they replied.
- Use parallel.ai's task API (alongside LinkedIn scrapers and Prospeo) to run unstructured research — website, company news, tech installs — that's more detailed than Clay and can be triggered by a webhook on positive reply.
- Use Sculptor inside Clay, Claude Code, or Codex to pull insights out of the post-reply enrichment table so you discover signals you never hypothesized (e.g., learning the real driver was "has a sales ops person" rather than AE-to-SDR ratio).
- Before layering signals at all, fix the offer — many "signal wins" disappear if you just have an offer that's easy to say yes to.
