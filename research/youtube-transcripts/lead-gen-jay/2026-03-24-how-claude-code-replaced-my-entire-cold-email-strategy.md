## Key takeaways
- Building custom micro-SaaS lead magnets in Claude Code (one app per client vertical, ~1 hour of build time) drove ~4% average reply rates even in previously "impossible" niches like AI automation agencies and cybersecurity firms.
- The prior make.com "perceived value magnet" approach lifted a cold-email-selling-cold-email campaign from 0.5% to ~3%, but AI-detection fatigue killed it and forced the shift to actual micro-SaaS deliverables.
- Concrete client example: an AI automation agency's micro-SaaS scans a prospect's site to detect their stack (GoHighLevel, Shopify, HubSpot) and generates one-click-import n8n automations tailored to those tools, then the cold email offers to "build a couple automations you can import in one click."
- Email 1 uses the pattern "Hey [name], I noticed you're running [tool] - would it be okay if I spent some time and put together a few automations for you? Just say yes and I'll get started"; Email 2 sends the prefilled app URL with the deliverable already generated.
- Idea generation prompt requires: 5 micro-SaaS ideas, input = URL or company name only, value delivered in under 60 seconds, plus a 1-hour Claude Code MVP build plan - Jay recommends running the prompt across Claude Opus 4.6, GPT, and Gemini to compare ideas before building.
- For non-technical users, Bolt.new and Lovable can produce the same apps in-browser in ~30 minutes with no terminal, but hit ceilings on complex backend logic and cost $20-50/month; Claude Code + Vercel + Supabase (both free tiers) gives more control and lower ongoing cost.
